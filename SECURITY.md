# Security Policy

agentos injects content into AI coding agents' contexts and modifies their
configuration — we treat that position seriously. The full threat model,
including what we protect against and how, is documented in
[docs/03-security.md](docs/03-security.md).

## Reporting a vulnerability

Email **khubaibnazeer8@gmail.com** with a description and reproduction steps.
Please do not open a public issue for security reports. You will get a
response within 72 hours. If the report is valid, a fix ships before any
public disclosure, and you will be credited (unless you prefer otherwise).

## Supported versions

Pre-1.0: only the latest release receives fixes.

## Verifying releases

Every release binary ships with a `SHA256SUMS.txt` and Sigstore build
provenance generated by CI (never from a maintainer's machine). Verify with:

```
gh attestation verify <downloaded-file> --repo Khubaib7-del/agentos
```

## Design commitments

- Local-first: no telemetry, no network calls at runtime
- Fail-open hooks: an agentos bug must never block your agent
- Secret redaction on every state write
- Dry-run by default for anything that touches other tools' configs
